# 2. Объяснение работы алгоритма
   
Алгоритм решает дробный рюкзак: можно брать предметы целиком или частями.

В коде он реализован в два логических шага, отражённых прямо в структуре:

**Шаг 1 — сортировка по удельной стоимости**

for (auto [w, v] : items)
    sorted.push_back({(double)v / w, {w, v}});
sort(sorted.begin(), sorted.end(), greater<>());


Каждый предмет превращается в тройку (стоимость/вес, вес, стоимость) и сортируется по убыванию отношения стоимость/вес.
Это значит, что сначала будут рассматриваться предметы, которые дают больше выгоды на единицу веса.

**Шаг 2 — наполнение рюкзака**

if (used + w <= capacity) {
    totalValue += v;
    used += w;
} else {
    int left = capacity - used;
    totalValue += ratio * left;
    break;
}


Программа перебирает отсортированные предметы и для каждого:
- Если он вмещается целиком — кладёт его полностью.
- Если не вмещается — берёт только столько веса, сколько осталось, и добавляет пропорциональную стоимость (ratio * оставшееся место).
После этого работа завершается (break), потому что рюкзак полностью заполнен.

# 3. Сложность по big O: 
O(n log n)
Обоснование:
 - Цикл создания массива с удельной стоимостью выполняется один раз для каждого предмета - тогда n итераций (O(n))
 - Сортировка (sort(sorted.begin(), sorted.end(), greater<>());) сортирует массив. Мы сравниваем каждый предмет с каждым и получается, что сложность O(n log n)
 - заполнение рюкзака идет по одному шагу, значит O(n)
Сложность исходит из самой сложной части кода - поэтому O(n log n)

# Ответ на контрольный вопрос №3:

Разница между PTAS и FPTAS заключается в том, от чего зависит время работы алгоритма при заданной точности приближения.

PTAS — Polynomial-Time Approximation Scheme
Для любого фиксированного ε > 0 (например, ε = 0.1, 0.01 и т.д.) алгоритм находит решение с гарантированной точностью.
Время работы — полиномиальное от размера входа, но только если ε фиксировано.
Однако степень полинома может зависеть от 1/ε.


FPTAS — Fully Polynomial-Time Approximation Scheme
Тоже даёт (1 ± ε)-приближение для любого ε > 0.
Но теперь время работы — полиномиальное и от n, и от 1/ε одновременно.
